`Miso.Html.Element` provides a constructor for every standard HTML element. All constructors follow a consistent naming convention and type signature.

## Naming convention

Every element constructor is the lowercase HTML tag name followed by an underscore. The underscore avoids conflicts with Haskell keywords (`data_`, `map_`, `type_`, etc.) and Prelude functions (`head_`, `main_`).

```haskell
div_  -- <div>
span_ -- <span>
p_    -- <p>
```

## Type signatures

Most elements accept attributes and children:

```haskell
div_ :: [Attribute action] -> [View model action] -> View model action
```

Void elements (those that can never have children in HTML) accept only attributes and return a `View` directly:

```haskell
input_  :: [Attribute action] -> View model action
img_    :: [Attribute action] -> View model action
br_     :: [Attribute action] -> View model action
hr_     :: [Attribute action] -> View model action
meta_   :: [Attribute action] -> View model action
link_   :: [Attribute action] -> View model action
source_ :: [Attribute action] -> View model action
track_  :: [Attribute action] -> View model action
embed_  :: [Attribute action] -> View model action
wbr_    :: [Attribute action] -> View model action
col_    :: [Attribute action] -> View model action
```

Two elements have a unique signature because they wrap raw text directly:

```haskell
style_  :: [Attribute action] -> MisoString -> View model action
script_ :: [Attribute action] -> MisoString -> View model action
```

The doctype pseudo-element takes no arguments:

```haskell
doctype_ :: View model action
```

<Warning>
`style_` and `script_` accept raw text and do **not** escape it. Only pass trusted content. Untrusted content may lead to CSS injection or XSS.
</Warning>

## Low-level constructor

All element functions are defined in terms of `nodeHtml`:

```haskell
nodeHtml :: MisoString -> [Attribute action] -> [View model action] -> View model action
nodeHtml nodeName = node HTML nodeName
```

Use `nodeHtml` to build custom or non-standard HTML elements.

<Note>
`form_` used with `onSubmit` prevents the browser from navigating. `onSubmit` sets `preventDefault = True` by default.
</Note>

```haskell
data Action = UpdateName MisoString | Submit

loginForm :: MisoString -> View model Action
loginForm name = form_ [ onSubmit Submit ]
  [ fieldset_ []
    [ legend_ [] [ "Sign in" ]
    , label_ [ for_ "username" ] [ "Username" ]
    , input_
      [ id_ "username"
      , type_ "text"
      , value_ name
      , onInput UpdateName
      , placeholder_ "Enter username"
      ]
    , button_ [ type_ "submit" ] [ "Sign in" ]
    ]
  ]
```

`Miso.Html.Element` re-exports `svg_` from `Miso.Svg.Element`:

```haskell

```

For all other SVG elements, import `Miso.Svg`.

## Nested elements example

```haskell
card :: MisoString -> MisoString -> View model action
card title_ content_ = div_ [ class_ "card" ]
  [ div_ [ class_ "card-header" ]
    [ h2_ [ class_ "card-title" ] [ text title_ ] ]
  , div_ [ class_ "card-body" ]
    [ p_ [] [ text content_ ] ]
  , div_ [ class_ "card-footer" ]
    [ button_ [ class_ "btn" ] [ "Read more" ] ]
  ]
```

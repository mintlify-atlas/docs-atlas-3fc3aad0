---
title: "HTML properties"
description: "DOM properties and attributes in Miso.Html.Property"
---

`Miso.Html.Property` provides combinators for setting DOM properties and attributes on virtual DOM nodes. All functions produce an `Attribute action` value.

## How properties work

```haskell
id_ :: MisoString -> Attribute action
```

Properties are passed as the first argument to any element constructor:

```haskell
div_ [ id_ "main", class_ "container" ] [ text "Hello" ]
```

## Properties vs attributes

Miso distinguishes between DOM *properties* (set on the JavaScript object) and HTML *attributes* (set as strings in the markup). Most combinators in this module set DOM properties directly — for example, `value_` sets the `value` property on an input element, which controls the *current* value rather than the initial attribute.

The underlying constructors:

```haskell
-- Set a string property on the DOM node
textProp :: MisoString -> MisoString -> Attribute action

-- Set a boolean property on the DOM node
boolProp :: MisoString -> Bool -> Attribute action

-- Set a numeric property on the DOM node
doubleProp :: MisoString -> Double -> Attribute action
```

These are from `Miso.Property` and you can use them directly for properties not covered by the named combinators.

## Class and identity

```haskell
-- Set the className property (both are equivalent)
class_     :: MisoString -> Attribute action
className  :: MisoString -> Attribute action

-- Set multiple space-separated classes from a list
classes_   :: [MisoString] -> Attribute action

-- Set classes conditionally
classList_ :: [(MisoString, Bool)] -> Attribute action

-- Set the id property
id_        :: MisoString -> Attribute action
```

```haskell
div_
  [ id_ "sidebar"
  , classList_
    [ ("sidebar", True)
    , ("sidebar--collapsed", isCollapsed)
    , ("sidebar--dark", isDark)
    ]
  ]
  [ text "Sidebar content" ]
```

<Note>
`class_` and `className` are identical — both set the `className` DOM property and both ultimately use the `ClassList` attribute constructor. `classList_` enables conditional class application without string concatenation.
</Note>

<Note>
`disabled_` takes no argument — it always sets `disabled` to `True`. To conditionally disable an element, use `optionalAttrs`:

```haskell
button_ (optionalAttrs isDisabled [ disabled_ ]) [ "Submit" ]
```
</Note>

```haskell
data Action = UpdateEmail MisoString | UpdatePassword MisoString | Submit

loginForm :: MisoString -> MisoString -> Bool -> View model Action
loginForm email password isLoading = form_ [ onSubmit Submit ]
  [ label_ [ for_ "email" ] [ "Email" ]
  , input_
    [ id_ "email"
    , type_ "email"
    , value_ email
    , onInput UpdateEmail
    , placeholder_ "you@example.com"
    , required_ True
    , autocomplete_ True
    ]
  , label_ [ for_ "password" ] [ "Password" ]
  , input_
    [ id_ "password"
    , type_ "password"
    , value_ password
    , onInput UpdatePassword
    , minlength_ "8"
    ]
  , button_ ([ type_ "submit" ] ++ [ disabled_ | isLoading ]) [ "Log in" ]
  ]
```

## Data attributes

Set arbitrary `data-*` attributes:

```haskell
data_ :: MisoString -> MisoString -> Attribute action
```

```haskell
-- Renders as: data-user-id="42"
div_ [ data_ "user-id" "42" ] []

-- Renders as: data-component="card"
div_ [ data_ "component" "card" ] []
```

## ARIA attributes

Set arbitrary `aria-*` attributes:

```haskell
aria_ :: MisoString -> MisoString -> Attribute action
```

```haskell
-- aria-label
button_ [ aria_ "label" "Close dialog" ] [ "×" ]

-- aria-expanded
nav_ [ aria_ "expanded" "true" ] [ ]

-- aria-hidden
span_ [ aria_ "hidden" "true", class_ "icon" ] [ ]
```

## Inline styles

Inline styles are set using the `Styles` attribute constructor (via `Miso.Property.style_` or directly). Styles are a `Map MisoString MisoString`:

```haskell
import qualified Data.Map.Strict as Map

coloredBox :: MisoString -> View model action
coloredBox color = div_
  [ Styles (Map.fromList
    [ ("background-color", color)
    , ("width", "100px")
    , ("height", "100px")
    , ("border-radius", "4px")
    ])
  ]
  []
```

During server-side rendering, styles are serialized as: `style="background-color:red;width:100px;"`.

## Namespace

```haskell
xmlns_ :: MisoString -> Attribute action
```

Used when embedding SVG or MathML inline:

```haskell
svg_ [ xmlns_ "http://www.w3.org/2000/svg", width_ "100", height_ "100" ] []
```

## Combining multiple properties

```haskell
fullFeaturedInput :: MisoString -> View model action
fullFeaturedInput currentVal = input_
  [ id_ "search"
  , class_ "input input--search"
  , type_ "search"
  , value_ currentVal
  , placeholder_ "Search…"
  , name_ "q"
  , autocomplete_ True
  , autofocus_ True
  , required_ True
  , aria_ "label" "Search the site"
  , data_ "testid" "search-input"
  ]
```
